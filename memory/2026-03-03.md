## 🔄 17:51 UTC更新 - MemU配置深度分析

### 技术问题确认
**核心问题**：SQLModel类型映射限制
- MemU的`MemoryItem`模型包含`embedding: list[float] | None`字段
- SQLModel无法自动映射Python `list`类型到SQLAlchemy列
- 影响：SQLite和PostgreSQL存储都失败

### MemU现有解决方案（但有问题）
**SQLite模型设计**：
```python
# SQLiteMemoryItemModel已尝试解决：
embedding_json: str | None = Field(...)  # 实际数据库列

@property
def embedding(self) -> list[float] | None:  # property覆盖父类字段
    return json.loads(self.embedding_json) if self.embedding_json else None
```
**问题**：SQLModel在模型合并时仍尝试映射父类的`list[float]`字段

### 测试结果总结
1. ✅ **内存存储（inmemory）**：可以初始化，但API调用需要`memory_categories`参数
2. ❌ **SQLite存储**：SQLModel类型映射失败（`ValueError: <class 'list'> has no matching SQLAlchemy type`）
3. ❌ **PostgreSQL存储**：缺少pgvector扩展（`extension "vector" is not available`）

- **效果**：永久解决存储问题

**方案B**：使用内存存储 + 修复API调用

2. **修改__annotations__覆盖** (❌失败):

### 根本问题
**SQLModel架构限制**:
- SQLModel在创建表时扫描所有父类的字段注解
- `MemoryItem.embedding: list[float]` → SQLModel尝试映射但失败
- 即使子类覆盖，父类字段仍被扫描
- SQLModel不支持`list[float]`到数据库列的自动映射
