**原T01命令问题**:
```bash
cd /root/.openclaw/workspace/tasks/T01 && PATH=/root/.nvm/versions/node/v22.22.0/bin:$PATH /usr/bin/python3 main.py t-day >> t01_daily_scoring.log 2>&1
```

**问题点**:
1. **PATH变量扩展**: `$PATH`在cron环境中可能为空或未正确扩展
2. **命令复杂度**: 单行命令包含多个操作（cd、PATH设置、Python执行、重定向）
3. **错误处理缺失**: 失败时无明确错误输出（MAILTO=""丢弃错误）

**包装器成功原因**:
1. **环境隔离**: 脚本在独立bash环境中执行，环境变量正确继承
2. **路径处理**: 脚本内部处理工作目录切换，避免`cd`命令失败风险
3. **错误可见**: 脚本输出到日志文件，便于调试

### **✅ 验证的解决方案**
1. **全局环境变量设置** (已验证有效):
   ```bash
   SHELL=/bin/bash
   PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/root/.nvm/versions/node/v22.22.0/bin
   HOME=/root
   MAILTO=""
   ```
2. **shell脚本包装器** (已验证有效):
   - 将复杂命令封装到shell脚本中
   - 脚本内部处理环境设置和错误处理
