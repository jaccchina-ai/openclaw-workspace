### 每日启动流程

2. **加载内存**: 读取最近2天的内存文件（YYYY-MM-DD.md）

### 心跳检查流程
1. **安全检查**: 注入扫描 + 行为完整性验证
2. **自我修复**: 日志分析 + 错误状态更新
3. **主动改进**: 扫描.learnings/，识别重复模式
4. **系统清理**: 磁盘/内存检查，清理建议

### 任务执行优先级

2. **阻塞问题**: 阻碍核心功能的问题（如MemU LLM配置）
3. **主动改进**: 构建自动化工具减少未来手动工作
4. **维护任务**: 磁盘清理、日志轮转、文档更新

## 🔧 自动化工具开发规范

### 新工具创建标准
1. **问题定义**: 明确解决的具体问题（如T99诊断）
2. **功能范围**: 单一职责，专注于一个痛点
3. **输入/输出**: 清晰的接口设计，便于集成
4. **错误处理**: 优雅降级，详细日志
5. **文档**: 使用说明和故障排除指南

### 测试与验证
1. **单元测试**: 核心功能独立测试
2. **集成测试**: 在真实环境中验证
3. **监控指标**: 成功率、执行时间、资源使用
4. **回滚机制**: 失败时恢复到之前状态

## 🚨 应急响应协议

### 连续失败处理
- **≥3次重复失败**: 必须创建自动化诊断工具
- **≥5次重复失败**: 立即停止手动修复，转向系统性解决方案
- **根本原因分析**: 识别模式而非症状

### 系统资源告警
- **磁盘 >80%**: 自动清理旧日志
- **内存 >90%**: 检查内存泄漏进程
- **CPU持续高负载**: 分析热点任务
